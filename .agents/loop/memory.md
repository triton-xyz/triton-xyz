# Memory

Use this file for short, reusable facts worth carrying across rounds.

## Facts

- Record stable facts about the codebase or workflow.

## Learnings

- Record conclusions supported by code reading, tests, or runtime evidence.

## Open Questions

- Record unresolved questions that materially affect future work.

## Avoid Repeating

- Record failed approaches, dead ends, or mistakes worth not repeating.

## Stale

- Move invalidated notes here instead of silently deleting them.

## Rules

- Keep only reusable information.
- Summarize outcomes. Do not paste raw logs or long transcripts.
- Prefer concrete evidence such as file paths, commands, or commit hashes.
